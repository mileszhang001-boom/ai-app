# 视觉优化指令 · 第三轮（配色专项）

> 本轮重点解决**配色规范遵守度不高**的问题。
> 所有色值都是精确的、可直接复制粘贴的，不需要任何"理解"或"发挥"。
> **规则：代码中不允许出现任何不在本文档中列出的颜色值。**

## 配色总原则

**每个组件有且只有一个"主题色系"，所有颜色从这个色系中取。不允许混搭、不允许自创色值。**

下面为每个组件列出**完整的色值清单**。实现时请逐一对照，不要凭记忆写颜色。

## 配色自查清单

实现完成后，请逐一对照检查：

**共性规则**（所有组件都必须遵守）：

- [ ] 卡片背景是渐变而非纯色（角度 140°-160°）
- [ ] 卡片边框统一为 `1px solid rgba(255,255,255,0.04)`
- [ ] 数字统一用白色渐变（上白下灰白），不是纯白
- [ ] 中文副标题统一 `rgba(255,255,255,0.70~0.75)`，不能更亮
- [ ] 中文辅助信息统一 `rgba(255,255,255,0.25~0.30)`，要足够淡
- [ ] 底部氛围光线高度 2px（不是 1px），宽度 40%-55%
- [ ] 所有英文标注统一 `text-transform: uppercase` + `letter-spacing: 3px`

**主题专属色规则**（每个组件的"主题色"只出现在以下位置）：

- [ ] 英文单位标注的文字色 → 使用主题色，透明度 0.35-0.50
- [ ] 分割线 → 使用主题色，透明度 0.15-0.20
- [ ] 底部氛围光线 → 使用主题色，透明度 0.35-0.50
- [ ] 背景光晕 → 使用主题色，透明度 0.10-0.18
- [ ] 类别标注（如 COUNTDOWN）→ 使用主题色，透明度 0.40-0.45

**禁止出现的颜色用法**：

- ❌ 任何硬编码的 `#ffffff` 用于英文标注（应该用主题色）
- ❌ 纯灰色 `rgba(255,255,255,0.5)` 用于英文标注（应该用带色调的主题色）
- ❌ 不同组件之间混用主题色（恋爱的粉色不能出现在宝宝的蓝色组件上）
- ❌ 卡片背景使用纯色（必须是渐变）
- ❌ 任何非设计系统中的色值

## 其他待修问题

### 闹钟数字字重

当前 10:45 看起来仍然很粗（像 font-weight: 700-800），需要改为 300。如果使用 Bebas Neue 字体（只有单一字重），那字体本身是可以的，但需要确认没有额外加粗。检查是否有 `font-weight: bold` 或 `font-weight: 700` 被应用。

### 新闻卡片每条新闻的布局

当前是"标签+时间"在一行、标题在下一行。应该改为：
- 第一行：[标签色块]  新闻标题（标签和标题同行）
- 第二行：时间（在标题正下方，极淡色）

### 预览页面背景色

预览页面的背景应该使用车机底色 `#0e1013`（带微妙蓝调的深色），不是纯黑 `#000000`。请检查当前 body 的 background 值。
