# 视觉优化指令 · 第二轮

> 基于实际预览截图的逐项分析。上一轮改了字体、背景渐变和光晕，效果有提升。本轮聚焦剩余问题。

---

## 问题 1（致命）：放假倒计时仍然显示"已开始"

这个上一轮就提过，还没修。这是最刺眼的问题——数字区域显示中文"已开始"，完全破坏了设计语言。

**修复**：把 demo 的目标日期改为一个未来日期，比如 `2025-10-01`（国庆节）。如果日期已过需要有兜底逻辑，显示"0"而不是文字。核心数字区域永远只能是数字。

---

## 问题 2（严重）：四张情感卡片的布局需要区分——不能全部沉底

当前四张情感卡片全部采用了底部对齐（flex-end），内容挤在底部 30%，上方 70% 完全空白。这就是你说的"排版中心集中在底部"的根源。

**设计规范里有两种布局，不同卡片应该用不同布局**：

### 恋爱纪念日、宝宝成长 → 保持底部对齐，但不变

这两张卡片沉底是对的。上方的大面积留白配合光晕，营造"从暗色空间中浮现"的感觉。但当前效果之所以感觉"空"，是因为上方区域的光晕太弱、几乎看不到。

**修复**：加强上方光晕的可见度，让留白区域不是"空的黑色"而是"有氛围的暗色空间"。

```css
/* 恋爱卡片的主光晕 —— 提高透明度和尺寸 */
.widget-love .glow-primary {
  top: 10%; right: -5%;
  width: 220px; height: 220px;   /* 更大 */
  background: radial-gradient(circle, rgba(200,120,180,0.18) 0%, transparent 70%);
  /* 从 0.12 提到 0.18，更可见 */
  filter: blur(50px);   /* 更柔和 */
}

/* 宝宝卡片的光晕 */
.widget-baby .glow-primary {
  top: 15%; left: 40%;
  width: 220px; height: 220px;
  background: radial-gradient(circle, rgba(120,160,255,0.12) 0%, transparent 70%);
  /* 从 0.07 提到 0.12 */
  filter: blur(50px);
}
```

### 放假倒计时、暖橙纪念日 → 改为居中布局

这两张卡片不应该沉底，应该内容居中（略偏下）。居中布局上下都有留白，数字在卡片的"黄金位置"，视觉冲击力更强。

**修复**：

```css
/* 放假倒计时 + 暖橙纪念日：改为居中 */
.widget-holiday,
.widget-warm {
  display: flex;
  flex-direction: column;
  justify-content: center;    /* 从 flex-end 改为 center */
  align-items: center;
  text-align: center;
  padding: 48px 28px 32px;   /* 上方 padding 略大，让重心微偏下 */
}
```

这样四张卡片就有了两种布局节奏：恋爱和宝宝沉底（上方有光晕氛围），放假和暖橙居中（数字在视觉中心）。并排看的时候视觉节奏会更丰富，不会觉得"四张一模一样"。

---

## 问题 3（严重）：闹钟卡片信息层级过多

当前闹钟卡片从上到下有：NEXT ALARM 标题 + 日期 → 10:45 大字 → "2小时后提醒" → "还有 1小时 44分钟" → 底部 1h/2h/4h/8h 按钮。

这是 **5 层信息**，严重违反"最多 3 层"的设计原则。信息过载让卡片显得"杂"而非"精致"。

**修复**：砍到 3 层，只保留最核心的信息：

```
第 1 层：10:45（核心大字）
第 2 层：WEEKDAYS（英文标注，什么时候响）  
第 3 层：工作日闹钟（中文说明）
```

去掉"2小时后提醒"、"还有 1小时 44分钟"和底部的 1h/2h/4h/8h 快捷按钮。这些是功能交互，不是 glanceable 信息卡片应该承载的。

同时，10:45 的数字字重太粗了（看起来像 700-800），需要改轻：

```css
.widget-alarm .hero-number {
  font-weight: 300;    /* 改轻！当前太粗 */
}
```

## 问题 4（中等）：新闻卡片布局太空

当前新闻卡片只看到一条新闻标题（"人工智能在汽车领域取得新突破"），下方大片空白，再出现一个"汽车"标签。整体感觉信息密度太低、布局不紧凑。

**修复**：新闻卡片应该展示 3 条简短新闻，紧凑排列：

```
┌─────────────────────────┐
│ ▎每日速览      今日·3月7日 │   ← 顶部标题栏
│                         │
│ 科技  AI在汽车领域取得突破  │   ← 新闻1
│       2小时前            │
│ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─    │   ← 渐隐分割线
│ 汽车  小米汽车Q2交付量创新高│   ← 新闻2  
│       4小时前            │
│ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─    │
│ 财经  A股三大指数集体上涨   │   ← 新闻3
│       5小时前            │
│___氛围光线_______________ │
└─────────────────────────┘
```

每条新闻是：左侧标签 + 标题（单行截断） + 时间，三条之间用渐隐分割线隔开。卡片应该看起来信息充实但不拥挤。

## 问题 5（中等）：恋爱和宝宝的背景色差异太小

放大看，恋爱（玫瑰）和宝宝（靛蓝）的背景在深色下几乎分不出来，都是偏暗的灰色。翠绿和琥珀的差异反而更好。

**修复**：适度提高恋爱和宝宝背景渐变中有色部分的饱和度：

```css
/* 恋爱 · 玫瑰 —— 加强紫红调 */
.widget-love {
  background: linear-gradient(155deg, #1e0e24 0%, #261530 35%, #180e1e 100%);
  /* 中间色从 #1e1428 → #261530，更偏紫红 */
}

/* 宝宝 · 靛蓝 —— 加强蓝调 */
.widget-baby {
  background: linear-gradient(160deg, #141828 0%, #1a2038 40%, #15182a 100%);
  /* 中间色从 #1a1e2e → #1a2038，更偏蓝 */
}
```

不需要大幅度改变，只需要中间过渡色的饱和度提高一点，并排看就能明显感知"这是紫的，这是蓝的"。

---

## 问题 6（轻微）：中文副标题字号偏大

当前"每一天都算数"、"每一天都是新的奇迹"这些副标题看起来字号偏大（可能是 18-20px），和数字的对比不够极端。

**修复**：

```css
.subtitle-cn {
  font-size: 15px;     /* 确认是 15px，不是更大 */
  font-weight: 400;    /* Regular，不是 Medium 或 Bold */
}

.caption-cn {
  font-size: 12px;
  color: rgba(255,255,255,0.30);  /* 要足够淡 */
}
```

## 优化优先级

```
1. 放假倒计时换成未来日期（改数据即可）           → 1分钟
2. 放假+暖橙改为居中布局                        → 15分钟
3. 闹钟卡片砍到3层 + 数字改轻                    → 15分钟
4. 新闻卡片改为3条紧凑布局                       → 20分钟
5. 加强恋爱/宝宝背景色差异 + 加强上方光晕          → 10分钟
6. 中文副标题字号确认 15px                       → 5分钟
```

改完 1-3 就会有明显改善，四张卡片不再是"四个一样的底部堆叠"，而是两种布局节奏交替。
