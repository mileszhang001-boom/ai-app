# 颜色设计方案 — v0.4 配色升级

> 解决两个核心问题：① 工具卡可选配色太少太暗；② 纪念日卡蒙层与背景图脱节
> 更新日期：2026-03-23

---

## 一、问题诊断

### 1.1 工具卡颜色为何都"很压抑"

**根因在 `color-engine.js` 第 76-78 行：**

```javascript
// 当前（v0.3）— 背景色生成逻辑
var bg1 = hslToRgb(hsl.h, sat * 0.50, 0.15);  // 饱和度砍半, 亮度固定 0.15
var bg2 = hslToRgb(hsl.h, sat * 0.55, 0.17);  // 饱和度砍半, 亮度固定 0.17
var bg3 = hslToRgb(hsl.h, sat * 0.40, 0.10);  // 饱和度砍6成, 亮度固定 0.10
```

无论用户选什么颜色，背景亮度都被压到 **0.10-0.17**（接近纯黑），饱和度被砍掉 **40-60%**。结果：蓝色和紫色的背景几乎看不出区别，所有卡片都像一个色系。

### 1.2 纪念日卡蒙层问题

当前：底部叠加一个低透明度纯色渐变 → 图片细节透出太多 → 文字可读性差
参考：苹果 iOS 锁屏 / 小米自定义壁纸 → 高不透明度毛玻璃面板 + 从图片提取主色调

## 二、配色方案升级 — 三层配色架构

### 2.1 核心洞察：不同卡片需要不同的颜色策略

不是所有卡片都需要「满屏染色」。效率工具（日程/闹钟）需要简约干净，氛围场景（天气/音乐/新闻）需要沉浸感，纪念日卡需要与照片协调。因此采用**三层配色架构**：

```
┌─────────────────────────────────────────────────────────┐
│               三层配色架构 (Color Tier System)             │
├──────────┬──────────────┬──────────────────────────────────┤
│  层级     │ 背景处理      │ 颜色的作用                        │
├──────────┼──────────────┼──────────────────────────────────┤
│ 🧊 Clean  │ 固定黑/白     │ 用户选色 → 仅影响点缀元素          │
│ 🌊 Mood   │ 色彩渐变      │ 用户选色 → 驱动整体背景氛围         │
│ 🖼 Photo  │ 背景大图      │ 自动取色 → 毛玻璃面板              │
└──────────┴──────────────┴──────────────────────────────────┘
```

### 2.2 配色能力矩阵

| **日程** | 🧊 Clean | 固定浅白色 `#F0F4F8` | ✅ 选强调色 | 事件圆点、FAB按钮、"距下次会议"标签 | `#3B82F6` 蓝 |

### 2.3 各层级设计详解

#### 🧊 Clean 层 — 日程 & 闹钟

**设计理念：** 效率工具，用户扫一眼获取信息。背景保持中性（白或黑），不用颜色干扰信息层次。颜色仅用于**语义点缀**：区分事件类型、标记开关状态。

```
日程（Light Mode）                    闹钟（Dark Mode）
┌────────────────────┐              ┌────────────────────┐
│  白色底 #F0F4F8     │              │  纯黑底 #1A1A1C     │
│                    │              │                    │
│  09:00 🔵 晨会     │ ← 蓝色圆点    │  08:00  ●─────○    │ ← 绿色toggle
│  14:00 🟠 评审     │ ← 橙色圆点    │  12:30  ●─────○    │
│  17:30 ⚫ 周报     │ ← 深色圆点    │  07:00  ○─────○    │ ← 灰色(关闭)
│                    │              │                    │
│  [ 距下次会议 45分 ] │ ← 蓝色背景    │               (+)  │ ← 绿色FAB
└────────────────────┘              └────────────────────┘
```

**用户选色的实际效果：** 选了紫色 → 事件圆点变紫、FAB变紫、活跃标签变紫。背景永远不变。

**color-engine 调用方式：**
```javascript
// Clean 模式：只输出强调色变量，不输出背景渐变
var palette = computePalette(hex, 'clean');
// palette.cssVars 只包含:
// --dyn-accent, --dyn-accent-bg, --dyn-ring-stroke,
// --dyn-label-color, --dyn-glass-border
// 不包含 --dyn-bg（背景固定不变）
```

#### 🌊 Mood 层 — 天气 & 新闻 & 音乐

**设计理念：** 氛围场景，颜色是体验的一部分。选蓝色 → 深海蓝的天气卡；选紫色 → 星空紫的音乐卡。背景渐变由 color-engine 全权驱动。

**color-engine 调用方式：**
```javascript
// Mood 模式：输出完整调色板（背景 + 玻璃 + 强调 + 粒子）
var palette = computePalette(hex, 'mood');
// palette.cssVars 包含全部 12 个变量，包括 --dyn-bg
```

#### 🖼 Photo 层 — 恋爱 & 宝宝 & 假期

**设计理念：** 照片是主角，颜色从照片自动提取。用户不选色，而是选背景图。

**为什么不让用户选色？** 如果用户选了暖色日落图，又手动选了冷色蓝紫配色，底部面板和图片就会色调割裂。自动取色保证上下一体 — 这是苹果/小米壁纸的做法。

**音乐卡的特殊逻辑：** 默认从专辑封面提取主色调（类似 Photo 层逻辑），但用户可覆盖。优先级：`用户手动选色 > 封面取色 > 默认色（#E84393）`。

### 2.4 扩展预设色板：6 → 12 色

**适用于 🧊 Clean 层（选强调色）和 🌊 Mood 层（选主色）。🖼 Photo 层不展示色盘。**

**设计原则：**
- 12 色覆盖色环 6 大色相区（红/橙/黄/绿/蓝/紫），每区 2 色（一暖一冷或一深一浅）
- Mood 层：每个颜色经过 color-engine v2 处理后，背景渐变必须色感明确、互相可区分
- Clean 层：每个颜色作为强调色时，在白底/黑底上都清晰醒目

### 2.6 Color Engine v2 — 双模式 + 调优参数

**新增：双模式 API**

```javascript
// v0.4 — computePalette 支持两种模式
function computePalette(hex, mode = 'mood') {
  // mode = 'mood'  → 输出完整调色板（含背景渐变），用于天气/新闻/音乐
  // mode = 'clean' → 仅输出强调色变量（背景固定），用于日程/闹钟
}
```

**核心改动：提高背景亮度 + 保留更多饱和度（仅 mood 模式）**

```javascript
// v0.4 — 新背景色生成逻辑
var bg1 = hslToRgb(hsl.h, sat * 0.70, 0.20);  // 饱和度保留70%, 亮度 0.20（↑33%）
var bg2 = hslToRgb(hsl.h, sat * 0.75, 0.23);  // 饱和度保留75%, 亮度 0.23（↑35%）
var bg3 = hslToRgb(hsl.h, sat * 0.55, 0.13);  // 饱和度保留55%, 亮度 0.13（↑30%）
```

**对比（以 #4A90E2 天空蓝为例）：**

| 变量 | v0.3 输出 | v0.4 输出 | 视觉差异 |
|------|----------|----------|---------|
| bg1 | `hsl(210, 38%, 15%)` = #172332 | `hsl(210, 54%, 20%)` = #17334d | 更饱和，深海蓝感 |
| bg2 | `hsl(210, 42%, 17%)` = #1a2c3e | `hsl(210, 58%, 23%)` = #1a3d5c | 明显蓝调，有层次 |
| bg3 | `hsl(210, 30%, 10%)` = #121a22 | `hsl(210, 42%, 13%)` = #14222f | 暗部也有色感 |

**效果预期：** 用户选蓝色 → 看到的是「深海蓝」渐变而非「灰黑微偏蓝」；选绿色 → 看到的是「暗夜森林」而非「灰黑微偏绿」。每种颜色都有自己明确的色彩性格。

### 2.7 Glass Panel 增强（Mood 模式专属）

当前 `glassBg` 和 `glassBorder` 的透明度偏低，导致毛玻璃卡片不够明显：

```javascript
// v0.3
glassBg: 'rgba(' + c.r + ',' + c.g + ',' + c.b + ', 0.10)',
glassBorder: 'rgba(' + bright.r + ',' + bright.g + ',' + bright.b + ', 0.16)',

// v0.4 — 提高 glass 可见度
glassBg: 'rgba(' + c.r + ',' + c.g + ',' + c.b + ', 0.15)',      // 0.10 → 0.15
glassBorder: 'rgba(' + bright.r + ',' + bright.g + ',' + bright.b + ', 0.22)',  // 0.16 → 0.22
```

## 三、纪念日卡 — 毛玻璃蒙层方案

### 3.1 设计参考

| 平台 | 做法 | 效果 |
|------|------|------|
| **iOS 锁屏** | 壁纸 + `backdrop-filter: blur(40px)` + 半透明色层（从图片提取）+ 0.7~0.85 不透明度 | 文字区域呈现壁纸"高斯模糊"色调，文字清晰，有空间感 |
| **小米主题壁纸** | 壁纸 + 底部 30% 渐变遮罩（取图底部主色 + 85%~95% 不透明度）| 底部区域近乎纯色，与壁纸自然过渡 |

### 3.2 实现方案（H5 模板内）

```css
/* 底部内容面板 — 毛玻璃效果 */
.anniversary-content-panel {
  position: absolute;
  bottom: 0;
  left: 0;
  right: 0;
  height: 40%;  /* 卡片底部 40% */
  border-radius: 32px 32px 0 0;
  overflow: hidden;

  /* 核心：双层叠加 */
  background: var(--panel-tint);              /* 从图片取色的半透明底色 */
  backdrop-filter: blur(32px) saturate(1.5);  /* 高斯模糊 + 增加饱和度 */
  -webkit-backdrop-filter: blur(32px) saturate(1.5);

  /* 玻璃边缘高光 */
  border-top: 1px solid rgba(255,255,255, 0.15);
  box-shadow: 0 -8px 32px rgba(0,0,0, 0.15);
}
```

### 3.3 背景图取色算法

纪念日卡不使用 color-engine 的用户选色，而是**自动从背景图提取主色调**生成面板色：

```javascript
/**
 * 从图片底部 30% 区域提取主色调
 * @param {HTMLImageElement} img — 背景图
 * @returns {string} — 'rgba(r,g,b,0.85)' 适合作为面板底色
 */
function extractPanelTint(img) {
  const canvas = document.createElement('canvas');
  const ctx = canvas.getContext('2d');
  const w = 80, h = 80;  // 缩小采样，性能优先
  canvas.width = w;
  canvas.height = h;

  // 只采样图片底部 30%
  const srcY = img.naturalHeight * 0.7;
  const srcH = img.naturalHeight * 0.3;
  ctx.drawImage(img, 0, srcY, img.naturalWidth, srcH, 0, 0, w, h);

  const data = ctx.getImageData(0, 0, w, h).data;

  // 加权平均（中心权重更高）
  let rSum = 0, gSum = 0, bSum = 0, weight = 0;
  for (let i = 0; i < data.length; i += 4) {
    const px = (i / 4) % w;
    const py = Math.floor((i / 4) / w);
    // 中心区域权重 2×，边缘 1×
    const cx = Math.abs(px / w - 0.5), cy = Math.abs(py / h - 0.5);
    const pw = 1 + (1 - cx * 2) * (1 - cy * 2);
    rSum += data[i]     * pw;
    gSum += data[i + 1] * pw;
    bSum += data[i + 2] * pw;
    weight += pw;
  }

  const r = Math.round(rSum / weight);
  const g = Math.round(gSum / weight);
  const b = Math.round(bSum / weight);

  // 将提取的颜色压暗（确保文字对比度）
  // HSL 空间：亮度不超过 0.35，饱和度保留 80%
  const hsl = rgbToHsl(r, g, b);
  const panelL = Math.min(hsl.l, 0.35);
  const panelS = hsl.s * 0.80;
  const panelRgb = hslToRgb(hsl.h, panelS, panelL);

  return `rgba(${panelRgb.r}, ${panelRgb.g}, ${panelRgb.b}, 0.88)`;
}
```

## 四、颜色安全性保障

### 4.1 "任何颜色都好看" 的约束

color-engine 必须保证：无论用户输入什么 hex，输出的卡片都可用。关键约束：

| 检查项 | 规则 | 代码保障 |
|--------|------|---------|
| 文字可读性 | 白色文字 vs 背景：对比度 ≥ 4.5:1 (WCAG AA) | bg 亮度上限 0.25 |
| 色感可辨识 | 背景饱和度 ≥ 0.15 | `sat = Math.max(sat, 0.20)` |
| 不刺眼 | 背景亮度不超标 | bg 亮度上限 0.25，glass 透明度 ≤ 0.20 |
| 玻璃卡片可见 | glassBg 不能与 bg 完全融合 | glassBg 亮度 ≥ bg 亮度 + 0.05 |

### 4.2 极端颜色保护

```javascript
// 纯白 #FFFFFF → 保护：使用 hue=210（蓝）, sat=0.20 的默认值
if (hsl.l > 0.95) { hsl.h = 210; sat = 0.30; }

// 纯黑 #000000 → 保护：使用 hue=210, sat=0.20
if (hsl.l < 0.05 && hsl.s < 0.05) { hsl.h = 210; sat = 0.30; }

// 灰色系（饱和度极低）→ 保护：给一点蓝灰调
if (sat < 0.10) { hsl.h = 210; sat = 0.15; }
```
