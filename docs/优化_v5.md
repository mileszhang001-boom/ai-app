# AI 创意工坊 · 研发交付文档

> 基于 Pencil 终稿标注，按「静态UI / 集成能力 / AI服务」三类拆解，供前端、后端、AI 团队对齐。

**图例：**  🖼 静态UI（纯前端布局/样式）　🔗 集成能力（需对接外部服务/SDK）　🤖 AI服务（需调用大模型）

---

## 一、全链路与分工总览

```
场景宫格 ──→ 配置面板 ──→ [AI生成] ──→ 预览 ──→ 微调（复用面板）──→ [同步车机]
   │                        🤖              🖼          🤖🖼            🔗
自由输入 ──→ [AI直接生成] ────────────────→ 预览
   🖼              🤖
```

| 环节 | 类型 | 说明 |
|------|------|------|
| 首页渲染（状态栏、Header、九宫格、输入栏） | 🖼 | 纯静态布局 |
| AI推荐标签（"AI推荐"显示在哪张卡片上） | 🤖 | 根据时间/季节/用户历史决定推荐哪个场景 |
| 配置面板渲染 | 🖼 | 按场景加载对应配置项，纯前端 |
| 城市自动定位 | 🔗 | 调用系统定位 API |
| 风格切换→配色推荐语更新 | 🤖 | 规则引擎或 LLM 生成推荐理由 |
| "优化表达"按钮 | 🤖 | 自然语言 → 结构化配置映射 |
| 卡片生成 | 🤖 | 核心 AI 生成，输入配置参数，输出卡片 JSON/HTML |
| 生成思考动效 | 🖼 | 前端动画，等待 AI 返回 |
| 生成摘要文案 | 🤖 | AI 返回卡片同时返回一句摘要 |
| 卡片内智能内容 | 🤖🔗 | AI 生成文案 + 拉取实时数据（天气 API、日历 API 等） |
| 微调时 AI 建议 | 🤖 | 基于当前卡片配置生成一条场景化建议 |
| 实时预览更新 | 🖼🤖 | 配置变更 → 局部重新生成 → 前端渲染 |
| 同步到车机 | 🔗 | 调用车机推送 SDK |

#### 自由输入行拆解

| 元素 | 类型 | 规格 |
|------|------|------|
| 标签 | 🖼 | "还想调什么？" 13px #6B7280 |
| 输入框 | 🖼 | fill_container × 40px, r10, #F8F9FC, 1px #E5E7EB 描边, placeholder "如：简洁一点，暖色调" |
| 优化表达按钮 | 🖼🤖 | fit_content × 40px, r10, #F0F4FF, "优化表达" 13px medium #4A6CF7 |

> 🤖 **"优化表达"逻辑：** 用户输入自然语言 → 调 LLM → 返回结构化配置变更。
> 输入：`{ text: "简洁一点，暖色调", currentConfig: {...} }`
> 输出：`{ changes: { infoLevel: "simple", colorScheme: "warm-orange" }, feedback: "✨ 已将信息量调为简洁，配色切换为暖橙" }`
> 前端接收后自动更新对应配置项 + Toast 反馈。

#### 卡片预览 🖼🤖

| 属性 | 值 |
|------|------|
| 容器 | fill_container, padding 8 40, 居中 |
| 卡片尺寸 | 260×434 (比例 664:1110), r20 |
| 卡片背景 | 渐变 #D4E5FF→#EEF2FF (示例，由 AI 生成结果决定) |

> 输入：`{ scene: "weather", style: "minimal", config: { city: "上海", infoLevel: "detailed", colorScheme: "blue" } }`

#### 操作按钮区 🖼

| 按钮 | 规格 | 行为 |
|------|------|------|
| "✏️ 微调效果" (主) | fill × 50px, r14, 渐变填充, 16px bold 白字, gap 8 带 pencil 图标 | 点击 → 弹出配置面板（复用 Screen 2 面板，配置项填入当前值）|
| "同步到车机 →" (次) | fill × 50px, r14, 白底, 1.5px #4A6CF7 描边, 16px medium #4A6CF7, arrow-right 图标 | 🔗 点击 → 调用车机推送 SDK |

#### 生成过程动效 🖼

```
用户点击 "✨ 生成卡片"
   ↓
展示 Loading 动效（前端）：
   "✨ AI正在创作…"
   → 理解需求 ✓ (300ms)
   → 匹配风格 ✓ (300ms)
   → 生成布局 ✓ (300ms)
   → 填充数据 ✓ (300ms)
   ↓
AI 返回结果 → 卡片淡入 + 缩放出场 (500ms)
```

> 总时长 1–1.5s，前端动效与 AI 请求并行，AI 返回后立即切换到结果。不超过 2 秒。

## 三、AI 服务接口清单

| # | 接口名称 | 触发时机 | 输入 | 输出 | 耗时要求 |
|---|---------|---------|------|------|---------|
| 1 | 场景推荐 | 首页加载 | 当前时间、季节、用户历史 | 推荐 sceneId | < 200ms |
| 2 | 配色推荐语 | 切换风格时 | 当前风格名 | 一句推荐文案 | < 300ms |
| 3 | 优化表达 | 点击"优化表达"按钮 | 用户输入文本 + 当前配置 | 结构化配置变更 + 反馈文案 | < 1s |
| 4 | 卡片生成 | 点击"生成卡片"/"应用修改" | 场景类型 + 全量配置参数 | 卡片 HTML + 摘要 + 洞察文案 | < 2s |
| 5 | 自由创作 | 首页发送按钮 | 用户自由文本 | 卡片 HTML + 摘要 | < 3s |
| 6 | 微调建议 | 进入微调页 | 当前配置 + 时间上下文 | 一条建议文案 + 操作按钮 | < 500ms |
| 7 | 局部更新 | 微调面板中切换配置项 | 变更字段 + 当前卡片 | 更新后的卡片 HTML | < 1s |

## 六、AI 克制原则

1. 每个页面最多一条主动 AI 提示，不堆叠
2. 所有 AI 提示可忽略，不阻断主流程
3. AI 解释文字用辅助样式（11–12px 灰色），不抢焦点
4. 生成思考动效不超过 2 秒
5. AI 建议语气用"推荐""试试"，不用"应该""必须"
